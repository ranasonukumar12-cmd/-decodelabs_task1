# E-Commerce Sales Data Cleaning and Preparation

## 📌 Project Overview

This project focuses on cleaning, validating, and preparing an E-Commerce Sales dataset using Microsoft Excel. The primary objective was to improve data quality by handling missing values, removing inconsistencies, and ensuring the dataset is reliable for analysis and business reporting.

The project demonstrates practical data cleaning techniques commonly used in data analytics and business intelligence workflows.

---

## 🎯 Objectives

- Identify and handle missing values.
- Detect and remove duplicate records.
- Standardize inconsistent data entries.
- Improve data accuracy and completeness.
- Prepare the dataset for analysis and reporting.
- Enhance overall data quality and usability.

---

## 📊 Dataset Description

The dataset contains e-commerce transaction records, including:

- Order ID
- Order Date
- Customer ID
- Product Information
- Quantity Ordered
- Unit Price
- Payment Method
- Shipping Address
- Order Status
- Tracking Number
- Coupon Code
- Referral Source
- Total Sales Amount

---

## 🛠️ Data Cleaning Process

### 1. Missing Value Handling
- Identified missing values in the dataset.
- Filled or corrected incomplete records where applicable.

### 2. Duplicate Record Removal
- Checked for duplicate transactions.
- Removed redundant entries to maintain data integrity.

### 3. Data Standardization
- Standardized Referral Source values.
- Cleaned Payment Method entries.
- Corrected inconsistent Order Status values.
- Improved text formatting and consistency.

### 4. Data Validation
- Verified the accuracy of records.
- Ensured consistency across all columns.
- Checked for formatting issues and corrected them.
