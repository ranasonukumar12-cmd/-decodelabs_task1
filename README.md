# E-Commerce Sales Data Cleaning and Preparation

## 📌 Project Overview
This project demonstrates the process of cleaning, validating, and preparing an E-Commerce Sales dataset using Microsoft Excel. The objective was to improve data quality, eliminate inconsistencies, and ensure the dataset is suitable for analysis and business decision-making.

---

## 🎯 Objectives
- Identify and handle missing values.
- Detect and remove duplicate records.
- Standardize inconsistent data entries.
- Improve overall data accuracy and reliability.
- Prepare the dataset for reporting and analytics.

---

## 📊 Dataset Description
The dataset contains e-commerce transaction information, including:

- Order ID
- Order Date
- Customer ID
- Product Details
- Quantity Ordered
- Unit Price
- Payment Method
- Shipping Address
- Order Status
- Coupon Codes
- Referral Sources
- Tracking Information

---

## 🛠️ Data Cleaning Process

### 1. Missing Value Treatment
- Identified incomplete records.
- Filled missing Coupon Code values where applicable.

### 2. Duplicate Record Analysis
- Checked for duplicate transactions.
- Removed redundant records to maintain data integrity.

### 3. Data Standardization
- Standardized Referral Source entries.
- Corrected inconsistencies in Payment Method values.
- Unified Order Status naming conventions.

### 4. Data Validation
- Verified data consistency across columns.
- Ensured proper formatting and accuracy of records.

---

## ✅ Key Improvements
- Enhanced dataset completeness.
- Improved consistency across categorical fields.
- Reduced data redundancy.
- Increased reliability for business analysis and visualization.
